# Configure pedantic and strict Clippy linting

## Summary

Enable the most pedantic and strict Clippy lint configuration to enforce high code quality standards across the codebase.

## Changes Needed

Add to `Cargo.toml`:

```toml
[lints.rust]
unsafe_code = "forbid"
missing_docs = "deny"

[lints.clippy]
pedantic = { level = "deny", priority = -1 }
nursery = { level = "deny", priority = -1 }
cargo = { level = "deny", priority = -1 }
```

This enables:
- `clippy::pedantic` — opinionated, stricter-than-default lints
- `clippy::nursery` — experimental lints that are likely correct
- `clippy::cargo` — lints about Cargo.toml correctness
- `unsafe_code = forbid` — prevent any use of `unsafe`
- `missing_docs = deny` — require doc comments on public items

## Acceptance Criteria

- [ ] `cargo clippy --all-targets` passes with zero errors
- [ ] Lint check enforced in pre-push hook
- [ ] Any intentional exceptions use `#[allow(clippy::...)]` with a justifying comment
