# list: show all registered workspaces with TTL status

## Problem

Users have no way to see which workspaces are currently registered without manually reading `~/.config/worktree/workspaces.toml`.

## Proposed Changes

Add a `worktree list` subcommand that reads the workspace registry and prints each entry with:
- The workspace path
- When it was created
- How long until it expires (or that it has already expired), based on the configured TTL

If no TTL is configured, omit the TTL column.

### Human-readable output

```
3 workspace(s) registered (TTL: 7days):
  /Users/alice/workspaces/bold_turing    created 2d ago   expires in 5d
  /Users/alice/workspaces/quirky_hopper  created 8d ago   EXPIRED
  /Users/alice/workspaces/calm_dijkstra  created 1d ago   expires in 6d
```

When no workspaces are registered:
```
No workspaces registered.
```

### JSON output (--json flag)

```json
{
  "ttl": "7days",
  "workspaces": [
    { "path": "/Users/alice/workspaces/bold_turing", "created_at": "2026-01-01T00:00:00Z", "expired": false },
    { "path": "/Users/alice/workspaces/quirky_hopper", "created_at": "2025-12-28T00:00:00Z", "expired": true }
  ]
}
```

## Acceptance Criteria

- [ ] `worktree list` prints all registered workspaces in human-readable format
- [ ] TTL and expiry info is shown when TTL is configured
- [ ] `--json` flag emits structured JSON to stdout
- [ ] Empty registry prints a clear message
- [ ] 100% line coverage
