# Open worktree by owner/name with random branch name

## Summary

Allow opening a worktree by specifying only the repo `owner/name` (e.g. `worktree open worktree-io/runner`), without an issue URL or branch name. The worktree branch name should be randomly generated using the same `adjective_noun` format used for multi-repo group names (e.g. `bold_turing`, `calm_lovelace`).

## Motivation

Sometimes you want to quickly spin up a worktree for a repo without tying it to any specific issue — just to experiment, spike, or do ad-hoc work. Currently you must provide an issue URL or explicit branch name. Accepting bare `owner/name` with an auto-generated random name removes that friction and keeps the UX consistent with the multi-repo naming convention.

## Acceptance Criteria

* [ ] `worktree open owner/name` (no issue URL, no explicit branch) is accepted as a valid input
* [ ] The worktree branch name is generated using the existing `name_gen::generate_name()` (adjective_noun format)
* [ ] The repo is cloned/fetched if not already present locally
* [ ] The worktree is created and opened in the configured editor as usual
* [ ] Works with both `open` and `open-multi` commands
* [ ] The `worktree://` URL scheme supports this form as well
* [ ] Tests cover the new input parsing path
