# feat: centralize TOML templates as compile-time assets

## Summary

Currently the template strings for the three generated TOML files are scattered across the codebase as inline Rust string literals:

- `.worktree.toml` scaffold — `SCAFFOLD` constant in `src/repo_hooks_scaffold.rs` (19-line escaped string)
- `config.toml` default — generated inline by `Config::to_toml_with_comments()` in `src/config/ser.rs`
- `workspaces.toml` — no reference template exists

This makes them hard to read, edit, and syntax-highlight in an editor.

## Proposed Solution

1. Extract each template into a real `.toml` file under `assets/`:
   - `assets/worktree.toml`
   - `assets/config.toml`
   - `assets/workspaces.toml`

2. Create a central `src/templates.rs` module that embeds them at compile time via `include_str!()`:
   ```rust
   pub const WORKTREE_TOML: &str = include_str!("../assets/worktree.toml");
   pub const CONFIG_TOML:   &str = include_str!("../assets/config.toml");
   pub const WORKSPACES_TOML: &str = include_str!("../assets/workspaces.toml");
   ```

3. Update `src/repo_hooks_scaffold.rs` to use `crate::templates::WORKTREE_TOML` instead of the inline escaped string.

4. Add a test that `CONFIG_TOML` stays in sync with `Config::default().to_toml_with_comments()` to catch drift.

## Benefits

- Template files get TOML syntax highlighting and IDE validation
- Single place to find all generated file templates
- Compile-time guarantees — missing asset file = compile error
- Easier to review diffs when templates change
