# add prune command to remove expired worktrees

The `prune` function exists in `src/ttl/mod.rs` and the `WorkspaceRegistry` tracks created worktrees, but there is no CLI subcommand that calls `prune` to actually delete expired workspaces.\n\nA `worktree prune` (or similar) command should read `workspace.ttl` from config, call `ttl::prune` against the registry, and remove the expired worktree directories and their entries from the registry.
