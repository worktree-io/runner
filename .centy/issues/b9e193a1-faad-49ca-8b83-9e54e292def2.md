# Config: add inline TOML comments above each field when saving

When `Config::save()` writes `config.toml`, the file is generated by `toml::to_string_pretty` which produces no comments. Users opening the file have no guidance on what each key does or what values are valid.

Add a small doc comment above every property in the serialized TOML so the file is self-documenting.

## Expected output

```toml
# Command used to open the editor in a worktree (e.g. "code ." or "nvim .").
# Leave unset to inherit from the $EDITOR environment variable.
[editor]
command = "code ."

# Controls automatic behavior when a worktree is opened.
[open]
# Whether to launch the configured editor automatically on open. Default: true.
editor = true

# Shell scripts executed around the open command.
[hooks]
# Script run before the worktree is opened.
"pre:open" = "echo pre"
# Script run after the worktree is opened.
"post:open" = "echo post"
```

## Requirements

* Replace the `toml::to_string_pretty` call in `Config::save()` with a hand-built serializer (or a post-processing step) that prepends a `# …` comment line above each key/section
* Comments must match the doc comments already present on the struct fields in `src/config/mod.rs`
* The output must still round-trip cleanly through `toml::from_str::<Config>`
* Add or extend unit tests to assert the saved content contains the expected comment strings
