# Add a temp-folder flag to store worktrees in the OS temp directory

## Summary

Add a boolean `workspace.temp` config flag. When enabled, worktrees are rooted under the OS temp directory (via `std::env::temp_dir()`) instead of `~/worktrees/`.

## Motivation

- Worktrees in the OS temp dir are automatically cleaned up on reboot — ideal for short-lived/throwaway work
- Keeps the home directory free from ephemeral checkouts
- No path management required from the user; the runner picks the right temp dir per platform

## Proposed Change

```toml
[workspace]
temp = true
```

When `temp = true`, replace `dirs::home_dir().join("worktrees")` with `std::env::temp_dir().join("worktrees")`.  
When unset or `false`, behavior is unchanged. Defaults to `false`.

## Affected Code

- `src/config/mod.rs` — add `temp: bool` to `WorkspaceConfig` (with `skip_serializing_if` so it is omitted when false)
- `src/issue/paths.rs` — `bare_clone_path()` reads the flag and switches the base path accordingly
