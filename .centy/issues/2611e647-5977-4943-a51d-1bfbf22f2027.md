# Allow opening a worktree from inside an existing repo

Currently `worktree open` requires an explicit issue reference (`<REF>`) — a GitHub URL, `worktree://` deep link, or `owner/repo#N`.

## Proposal

Allow invoking `worktree open` (or a new subcommand) **from inside a git repository** with no arguments. When no `<REF>` is provided and the current directory is inside a git repo, the CLI should:

1. Detect the repository root and remote origin
2. Derive the owner/repo from the remote URL
3. Open/create a worktree for the current branch (or prompt for a branch/issue)

This enables a workflow where a user `cd`s into a cloned repo and runs `worktree open` to manage it as a worktree workspace without needing to look up the issue URL.

## Acceptance criteria

- `worktree open` with no `<REF>` inside a git repo detects the repo and opens the worktree
- Works with both HTTPS and SSH remote URLs
- Shows a clear error when run outside a git repo without a `<REF>`
- Existing `<REF>`-based behavior is unchanged
