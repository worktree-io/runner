# Add skip-hooks flag (CLI, lib, and web)

## Summary

Add the ability to skip hooks when opening a worktree, exposed across all three surfaces:

* **CLI**: `--no-hooks` flag on the `open` and `open-multi` commands
* **Lib**: a parameter/option in the relevant library functions so callers can opt out of hook execution
* **Web**: support the flag via the `worktree://` deep-link URL scheme (e.g. `?no-hooks=1`)

## Motivation

Sometimes users need to open a worktree without triggering pre/post hooks (e.g. in CI, scripting, or debugging). There is currently no way to bypass hooks without modifying config.

## Acceptance Criteria

* [ ] `worktree open <ref> --no-hooks` skips pre- and post-open hooks
* [ ] `worktree open-multi` supports the same flag
* [ ] Library exposes an option/param to skip hooks
* [ ] `worktree://` URL scheme supports a query param to skip hooks
* [ ] Tests cover the no-hooks paths
