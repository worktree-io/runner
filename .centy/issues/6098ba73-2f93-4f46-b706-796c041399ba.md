# Show freed space per folder after prune

## Summary

When running the `prune` command, display the amount of disk space freed for each folder/worktree that is removed, plus a total at the end.

## Motivation

Currently prune silently removes stale worktrees with no feedback on how much space was reclaimed. Users have no way to know whether pruning a particular folder was worth it or if there are larger culprits they should investigate manually.

## Acceptance Criteria

* [ ] Before removing each folder, calculate its size on disk
* [ ] After removal, print freed space for that folder (e.g. `pruned worktrees/issue-12 (freed 142 MB)`)
* [ ] Print a total freed space summary at the end of the prune run
* [ ] Human-readable units (B, KB, MB, GB) based on size
* [ ] No output change when nothing is pruned
