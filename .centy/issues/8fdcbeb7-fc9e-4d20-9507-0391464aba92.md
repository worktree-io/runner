# prune: structured and clearer output

## Problem

The current `worktree prune` output is minimal and not machine-readable:

- When nothing is pruned: `No expired workspaces found.` (printed to stderr)
- When workspaces are pruned: one line per path, then a summary count

## Proposed Changes

### Human-readable improvements

- **No expired workspaces**: include context such as how many workspaces were checked and what TTL is configured
- **Expired workspaces found**: show path, age, and configured TTL per entry; make the summary line more descriptive

### JSON output

Add a `--json` flag that emits a structured result:

```json
{
  "checked": 5,
  "pruned": [
    { "path": "/path/to/workspace", "expired_at": "2026-01-10T00:00:00Z" }
  ],
  "ttl": "7days"
}
```

This makes `worktree prune` composable with scripts and CI pipelines.

## Acceptance Criteria

- [ ] Human output is clear in both the empty and non-empty case
- [ ] `--json` flag emits valid JSON to stdout
- [ ] Exit code remains `0` in both cases (no breaking change)
