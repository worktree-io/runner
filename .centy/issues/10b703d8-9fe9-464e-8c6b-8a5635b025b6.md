# open-multi command: open one issue across multiple repos in a unified workspace

## Overview

Add a new `worktree open-multi` command that takes a single issue reference and a list of repositories, then opens all of them as worktrees under a single unified workspace folder.

## Motivation

Users working on an issue that spans multiple related repos (e.g., backend + frontend, or microservices) currently have to run `worktree open` separately per repo and open each folder individually. `open-multi` solves this by creating one shared workspace with all repos checked out together.

## Command Signature

```
worktree open-multi <issue> <owner/repo-a> <owner/repo-b> [<owner/repo-c> ...]
```

### Examples

```
worktree open-multi worktree-io/runner#42 worktree-io/runner worktree-io/landing
worktree open-multi https://github.com/acme/backend/issues/7 acme/backend acme/frontend
```

## Behavior

- Creates a unified workspace folder with a random human-friendly name (e.g., `~/workspaces/bold_turing/`)
- For each repo, clones (bare) if not already cached, then creates a worktree at `<workspace>/<repo-name>/`
- Branch name is derived from the issue (e.g., `issue-42`) and **always created in every repo** regardless of whether the issue exists there
- Editor is launched at the unified workspace root (not per-repo)
- TTL applies to the entire unified folder
- Per-repo hooks (`.worktree.toml`) run for each repo; group-level hooks (global config) run once for the workspace

## Workspace Layout

```
~/workspaces/bold_turing/
  runner/    ← worktree for worktree-io/runner
  landing/   ← worktree for worktree-io/landing
```
