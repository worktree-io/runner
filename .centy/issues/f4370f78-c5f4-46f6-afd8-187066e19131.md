# Workspace TTL: add a library for time-to-live management

Add a dedicated library crate (or module) that encapsulates all TTL logic for worktree workspaces — determining when a workspace has expired, pruning stale ones, and persisting TTL configuration.

## Motivation

Worktree workspaces can accumulate over time. A TTL mechanism lets users define a maximum age for workspaces and automatically clean up ones that are no longer needed.

## Requirements

* Define a `Ttl` type (e.g. a duration) that can be specified per-workspace or as a global default in config
* Store the workspace creation timestamp alongside the TTL value
* Provide a pure library function `is_expired(workspace: &Workspace, now: SystemTime) -> bool`
* Provide a `prune(workspaces: &[Workspace], now: SystemTime) -> Vec<&Workspace>` helper that returns expired workspaces
* Serialize/deserialize TTL config via serde (TOML-compatible)
* No side-effects in the core library — callers are responsible for actually deleting or closing workspaces

## Out of Scope

* CLI subcommand integration (tracked separately)
* Automatic background pruning daemon
