# test: add Docker-based integration testing

## Summary

The current test suite runs only on the host machine (macOS), making it difficult to verify behavior on Linux, validate git operations in a clean environment, and test cross-platform edge cases. Adding Docker-based integration tests would provide a reproducible, isolated environment to run the full test suite against a real Linux system.

## Proposed Solution

1. Add a `Dockerfile` (e.g. `docker/Dockerfile.test`) based on a minimal Rust + git image:
   - Install `cargo-llvm-cov` and any other test dependencies
   - Copy the workspace source into the container
   - Run `cargo test` (and optionally `cargo llvm-cov`) inside the container

2. Add a `docker-compose.yml` (or shell script `scripts/test-docker.sh`) that:
   - Builds the test image
   - Runs the full test suite and streams output back

3. Wire it into CI (GitHub Actions) as a separate `test-linux` job on every PR and push to `main`.

## Acceptance Criteria

- `docker compose run test` runs the full test suite successfully on Linux
- The Docker job is added to the existing CI workflow (`.github/workflows/`)
- All existing tests pass inside the container with no modifications to test logic
- The `Dockerfile` is minimal — no unnecessary layers or tooling

## Notes

- The pre-push hook's 100-line limit and 100% coverage requirement must still hold inside Docker
- Consider `rust:slim` or `rust:alpine` as the base image to keep build times reasonable
- `cargo-husky` hooks do not need to run inside the container (testing only)
