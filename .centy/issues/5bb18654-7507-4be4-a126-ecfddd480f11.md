# Sign app bundle with Developer ID for Gatekeeper-trusted distribution

## Problem

The `worktree-runner` install command currently ad-hoc signs the `WorktreeRunner.app` bundle (`--sign -`), which means Gatekeeper won't trust it on other users' machines.

## Goal

Enable proper signing with a **Developer ID Application** certificate so the app is Gatekeeper-trusted for distribution.

## Architectural Challenge

The app bundle is built dynamically at install time via `osacompile` (on the user's machine), which makes traditional pre-distribution notarization impossible. The AppleScript hardcodes the absolute path to the `worktree` binary.

## Proposed Solution

1. **Refactor the app bundle** to resolve the `worktree` binary via `PATH` at runtime instead of hardcoding the path at install time — this allows a single pre-built bundle to be distributed.
2. **Create a CI/release step** that:
   - Builds the bundle
   - Signs it with a Developer ID Application cert
   - Submits to Apple's notarization service (`xcrun notarytool`)
   - Staples the ticket (`xcrun stapler staple`)
   - Includes the notarized bundle in the release archive
3. The install command copies the pre-built bundle instead of building it with `osacompile`.
