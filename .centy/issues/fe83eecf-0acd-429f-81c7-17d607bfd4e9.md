# Remove --print-path flag from open command

The `--print-path` flag on the `open` subcommand prints the workspace path to stdout and returns early, skipping hooks and editor launch. This use case (scripting/cd) should be handled by a dedicated subcommand (e.g. `worktree path <issue>`) rather than a flag that silently changes the behavior of `open`.\n\n## What to remove\n- `print_path: bool` field from the CLI struct in `src/main.rs`\n- `print_path` parameter from `cmd_open` in `src/commands/open/mod.rs`\n- The early-return block gated on `print_path`\n\n## Tests\n- Remove or update any tests that pass `print_path: true`
