# feat: implement open-multi command for multi-repo unified workspaces

## Overview

Implement the `worktree open-multi` command that opens multiple repos as worktrees under a single unified workspace folder.

## Command Signature

Each argument is a repo+issue pair (shorthand like `acme/backend#7` or a full GitHub issue URL).

## Behavior

- Generates a random human-friendly workspace name (e.g., `bold_turing`)
- Creates `~/workspaces/<name>/` as the unified root
- For each repo: clones (bare) if not cached, creates worktree at `<root>/<repo>-<issue>/`
- Branch `issue-N` created in every repo
- Runs per-repo hooks for each worktree; global hooks run once
- Editor launched at unified workspace root
- TTL applies to the entire unified folder

## Files to Add

- `src/name_gen.rs` — random name generator
- `src/multi_workspace.rs` — multi-workspace creation logic
- `src/commands/open_multi.rs` — CLI command handler
