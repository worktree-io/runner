# feat(tmux): name session after repo, window after worktree

## Problem

When opening a worktree with `editor = "tmux"`, the runner creates a tmux session/window with no explicit name. This makes it hard to navigate between worktrees in tmux.

## Desired Behavior

Use the worktree path components to derive meaningful names:

- **Session name** = parent directory name (the repo name, e.g. `runner`)
- **Window name** = last path component (the worktree/branch name, e.g. `feature-login`)

If a session with that name already exists, attach to it and open a new window instead of creating a duplicate session. This naturally groups all worktrees of the same repo under one tmux session.

## Implementation Notes

In `src/opener/terminal.rs`, inside the `cmd_lower.starts_with("tmux")` branch:

1. Extract `session_name` from `path.parent().and_then(|p| p.file_name())`
2. Extract `window_name` from `path.file_name()`
3. Check if the session exists: `tmux has-session -t <session_name>`
   - If it exists → `tmux new-window -t <session_name> -n <window_name> -c <path> "sh <script>"`
   - If not → `tmux new-session -s <session_name> -n <window_name> -c <path> "sh <script>"`
4. Remove the current `$TMUX`-based branch (it becomes redundant)
