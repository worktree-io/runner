# Config: add `edit` subcommand to open config TOML in editor

Add an `Edit` variant to `ConfigAction` that resolves the config file path and opens it in the user's configured editor.

## Requirements

* Add `ConfigAction::Edit` with doc comment `/// Open the configuration file in the editor`
* Resolve the config path via `Config::path()?`; if the file does not exist, initialize it with `Config::default().save()?` first
* Open the file using the configured `editor.command` (reuse the existing opener logic)
* If no editor is configured, fall back to the `$EDITOR` environment variable, then error with a helpful message
