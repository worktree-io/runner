# Add --headless flag to skip opening editor/terminal

When `worktree` is invoked programmatically (e.g. from the `team` daemon's worker pool), opening an editor or terminal is not desired. A `--headless` (or `--no-open`) flag should skip the `opener::open_in_editor` / `opener::open_editor_or_terminal` step entirely, run the pre/post hooks, and exit with a meaningful exit code reflecting success or failure.

This allows `worktree <issue-ref> --headless` to be used as a fire-and-forget subprocess from a queue-based worker pool that only cares about the exit code.
