# Config: add website link comment at top of config file

When \`Config::save()\` writes \`config.toml\`, the file should include a comment at the very top pointing users to the project website for documentation and reference.

## Expected output

\`\`\`toml

# runner — https://worktree.io

[editor]
command = "code ."

[open]
editor = true

[hooks]
"pre:open" = "echo pre"
"post:open" = "echo post"
\`\`\`

## Requirements

- Prepend a \`# runner — <website URL>\` comment as the first line of the serialized TOML in \`Config::save()\`
- The comment must appear before any section headers or key-value pairs
- The output must still round-trip cleanly through \`toml::from_str::<Config>\`
- Add or extend unit tests to assert the saved content starts with the expected comment line
